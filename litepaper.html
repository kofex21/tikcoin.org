1. Overview

TIKcoin is a mechanism of trust: a clean, immutable ERC‑20 base with transparent distribution, public locks, and no admin keys. We focus on honesty, fairness and immutability: rules are defined at inception and respected over time. Utilities are built as external modules so the base never changes.

2. Problem

Too many tokens rely on opaque controls, arbitrary changes, and short‑term hype. This erodes trust and makes participants dependent on insiders. We aim to demonstrate that a minimal, verifiable foundation can compound trust long‑term.

3. Vision & Principles

  • Honesty. Publish what matters; keep verifiable records.
  • Fairness. Equal rules for all, enforced in code.
  • Immutability. Base ERC‑20 is final — no taxes, no burn mechanics, no blacklists, no owner functions.
  • Evolution as extensions. New utilities live in separate contracts/interfaces without breaking the foundation.
  • Measured stewardship. Treasury actions via multisig + timelock, with receipts.

6. Launch Plan

  1. Birth. Public channels, website, litepaper (this document). Early community formation.
  2. Foundation. Deploy audited base ERC‑20 + claim/vesting contracts. Publish addresses and locks.
  3. Presale. Per‑wallet cap; transparent reporting; bonuses (10/20/30%) are distributed from the 15% bonus sub‑pool and locked ≥12 months.
  4. Liquidity & TGE. Add initial LP (targeting measured pricing), publish LP tx and timelocked stabilization reserves.
  5. Listings. Start with DEX; pursue CEX when organic metrics justify it.

7. Security & Audits

  • Base token is minimal, using a well‑reviewed ERC‑20 implementation; no owner functions.
  • Independent audits in phases (at least two), including static analysis + manual review.
  • Public bug bounty after TGE; scope includes vesting/claims and ancillary modules.
  • Operational security: multisig signers, hardware wallets, timelock, monitored addresses.

8. Governance & Treasury

Treasury is controlled by a multisig + timelock. Initial signers are intended to include: founder, independent auditor partner, legal counsel (jurisdictional compliance), technical advisor, and a community representative. Threshold and addresses will be published pre‑TGE. All spends require on‑chain proposals with receipts. No spend can alter the base token rules.

9. Roadmap (high‑level)

  • Q3–Q4 2025. Finalize specs, audits (phase 1), presale infra, website growth, community milestones.
  • Q4 2025. TGE, initial LP, start of vesting/claims, audit phase 2, public bounty.
  • 2026. Utility modules, listings expansion, brand collaborations, measured treasury deployments.
